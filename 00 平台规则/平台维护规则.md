# 平台维护规则

本文档是给产品维护者看的，不是程序运行文件。目标是让后续新增、修改、删除单元时，有固定规则可遵循。

## 1. 文件夹分工

当前推荐结构：

```text
Swedish study/
  00 平台规则/
  01 音频整理/
  02 tools/
  audio/
  index.html
  common.css
  common.js
  practice-common.js
  sfi_b3_unit3.html
  sfi_practice_B_unit3.html
  b3_unit3_bank.js
```

各部分用途：

```text
00 平台规则
给维护者看的规则文档。

01 音频整理
放音频总台账、映射表、少量人工修正规则。

02 tools
放检查、生成、导出工具。

audio
放 mp3、audio-manifest.json、content-manifest.json。
不要改名为 03 audio，因为网页和工具里大量使用 audio/sv/... 这个路径。

根目录页面文件
放首页、单元学习页、练习页、题库文件和公共代码。
这些文件先不要移进编号文件夹，否则很多页面链接需要一起改。
```

## 2. 哪些文件可以编号

可以编号：

```text
00 平台规则/
01 音频整理/
02 tools/
```

暂时不要编号：

```text
audio/
index.html
common.css
common.js
practice-common.js
sfi_*.html
sfi_practice_*.html
*_unit*_bank.js
```

原因：这些文件会被网页直接访问，移动后容易造成链接失效。

## 3. 单元相关文件

每个学习单元通常有三类文件：

```text
单元学习页
例如：sfi_b3_unit3.html

练习页面
例如：sfi_practice_B_unit3.html

题库文件
例如：b3_unit3_bank.js
```

目前建议保持这个命名方式，不要随意改名。

## 4. 公共文件

这些文件是多个单元共享的：

```text
common.css
common.js
practice-common.js
```

修改这些文件会影响多个单元。改动前要特别谨慎，改完后至少测试 B3，必要时抽测 A/B/C 其他单元。

## 5. GitHub 上传规则

应该上传：

```text
00 平台规则/
01 音频整理/瑞典语学习-音频整理.xlsx
01 音频整理/b3_unit3_mapping_overrides.json
02 tools/
audio/
页面 HTML / JS / CSS
```

不要上传：

```text
.DS_Store
~$瑞典语学习-音频整理.xlsx
*.preview.json
*.before-*.json
B3_当前使用音频清单.xlsx
```

说明：`B3_当前使用音频清单.xlsx` 是临时检查报告，需要时可以重新生成，不是网页运行必需文件。

## 6. 当前 B3 的稳定规则

B3 当前采用的规则：

```text
单词页点击单词后播放对应音频。
播放结束后，高亮自动消失。

练习页第一次播放需要用户点击触发。
之后继续做题时，下一题可以自动播放。

练习页名词变形题可以显示多个空。
但 en/ett + 单数不定式 可以使用一条完整自然音频。

测验页不显示题型标签。
题型信息仍保留在数据中，方便以后恢复或统计。
```

## 7. 不要凭感觉删除文件

删除音频、manifest、页面文件前，先跑检查工具。

B3 检查命令：

```bash
python3 "Swedish study/02 tools/check_unit.py" --unit b3 --skip-index
```

只有确认不再使用，才删除。

# B4-B5 新单元复制演练清单

本文档用于正式开新单元前演练一次流程。目标不是马上写内容，而是确认“如果今天要做 B4/B5，我们不会漏步骤”。

## 1. 先确认是否真的适合复制 B3

B3 适合复制的情况：

```text
新单元也有学习页
新单元也有练习页
新单元也有基础模式和进阶模式
新单元也需要音频播放
新单元也会用 Excel 管理音频
```

如果新单元的练习逻辑完全不同，例如听写、长篇阅读、写作批改，就不要直接复制 B3 后硬改，应该先单独讨论结构。

## 2. 你开新单元前需要准备什么

你作为产品和内容负责人，先准备这些信息：

```text
新单元编号：例如 B4
新单元主题：例如 时间表达、交通、购物、工作等
学习页需要哪些板块
练习页需要哪些题型
是否沿用 B3 的基础/进阶模式
是否需要新增特殊题型
是否已有 mp3
是否允许复用旧音频
Excel 是否已经开始记录音频
```

如果暂时没有完整内容，也可以先给一个粗略范围。不要一边不确定内容，一边大量复制页面。

## 3. 我开工前需要检查什么

正式复制前，我先检查：

```text
B3 当前是否仍然 ERROR=0
公共文件 common.js / practice-common.js / common.css 是否稳定
首页是否已经有新单元入口
新单元目标文件名是否和现有文件冲突
audio/sv/ 下是否已有对应目录
Excel 里是否已有新单元音频记录
```

如果这些没确认，先不写新单元代码。

## 4. B3 复制到 B4 的文件清单

如果以 B3 作为模板生成 B4，通常复制：

```text
sfi_b3_unit3.html -> sfi_b4_unit4.html
sfi_practice_B_unit3.html -> sfi_practice_B_unit4.html
b3_unit3_bank.js -> b4_unit4_bank.js
```

新建：

```text
audio/sv/b4/unit4/vocab/
audio/sv/b4/unit4/dialogue/
audio/sv/b4/unit4/question/
audio/sv/b4/unit4/audio-manifest.json
audio/sv/b4/unit4/content-manifest.json
```

不复制：

```text
common.js
common.css
practice-common.js
02 tools/
01 音频整理/瑞典语学习-音频整理.xlsx
```

这些是共享文件或总台账。

## 5. B4 必须替换的关键词

复制后必须逐项替换：

```text
b3 -> b4
B3 -> B4
unit3 -> unit4
Unit3 -> Unit4
第三单元 -> 第四单元
sfi_b3_unit3 -> sfi_b4_unit4
sfi_practice_B_unit3 -> sfi_practice_B_unit4
b3_unit3_bank.js -> b4_unit4_bank.js
UNIT_BANK_B3 -> UNIT_BANK_B4
audio/sv/b3/unit3 -> audio/sv/b4/unit4
```

题目 id 也要统一：

```text
b3_vocab_...
b3_phrase_...
b3_dialog_...
b3_noun_...
```

改成：

```text
b4_vocab_...
b4_phrase_...
b4_dialog_...
b4_noun_...
```

## 6. 音频准备顺序

音频不要先乱放文件夹，顺序应该是：

```text
1. 在 Excel 里登记新单元音频
2. 标记是否主文件
3. 如果复用旧音频，写清楚沿用的原文件名
4. 如果新生成 mp3，放到新单元对应文件夹
5. 生成 manifest 预览文件
6. 检查没有缺失
7. 再替换正式 manifest
```

最容易出错的是：

```text
mp3 已经放了，但 Excel 没写
Excel 写了，但 manifest 没更新
manifest 指向了旧文件名
页面里 audioRef 和 manifest 里的 audioId 不一致
```

## 7. 练习题准备顺序

练习题不要先堆很多，先做最小可测试版本：

```text
基础模式 3-5 题
进阶模式 3-5 题
每种新题型至少 1 题
每题都能找到对应音频
```

最小版本跑通后，再批量增加题目。

如果新单元有特殊题型，先单独讨论：

```text
是否可以用现有 kind
是否需要新增 source
是否需要改 practice-common.js
是否会影响旧单元
```

## 8. 首页入口规则

新单元页面做好后，首页再加入口。

首页入口需要确认：

```text
学习页链接正确
练习页链接正确
单元标题正确
状态显示正确
没有指向临时页面
没有指向不存在文件
```

不要提前加一个会 404 的入口。

## 9. 本地检查

每次新单元至少跑：

```bash
python3 "Swedish study/02 tools/check_unit.py" --unit b4 --skip-index
```

目标：

```text
ERROR=0
没有缺失 mp3
题库能找到内容
内容能找到音频
```

如果有 warning，不一定立刻修，但要知道原因。

## 10. 本地测试

本地测试地址示例：

```text
http://127.0.0.1:8765/sfi_b4_unit4.html
http://127.0.0.1:8765/sfi_practice_B_unit4.html
```

至少测试：

```text
学习页能打开
单词音频能播放
对话音频能播放
基础练习第一题能播放
进阶练习第一题能播放
下一题能继续播放
切换模式不会播放旧题音频
测验页显示正常
```

## 11. 上传 GitHub 前检查

上传前确认只上传正式文件：

```text
学习页 html
练习页 html
题库 js
正式 audio-manifest.json
正式 content-manifest.json
mp3 文件
必要的规则文档
Excel 总台账
```

不要上传：

```text
.DS_Store
~$ 开头的 Excel 临时文件
*.preview.json
*.before-*.json
临时备份 html
```

## 12. GitHub 上传后测试

线上测试不用每次全量，但至少测：

```text
首页能进入新单元
学习页能打开
练习页能打开
1 个单词音频
1 个对话音频
1 个基础练习音频
1 个进阶练习音频
```

如果本地正常、线上异常，先强制刷新，再检查文件是否真的上传成功。

## 13. 什么时候才算新单元模板通过

满足这些条件，才算新单元结构通过：

```text
本地 ERROR=0
线上页面能打开
线上音频能播放
首页链接不 404
Excel 有记录
manifest 是正式文件
没有临时文件上传
你能根据清单说清楚这个单元的文件在哪里
```

## 14. 当前建议

下一次正式新增单元时，建议不要一次性做完整 B4。

推荐先做：

```text
B4 空壳页面
B4 最小题库
B4 少量测试音频
B4 manifest
本地跑通
线上跑通
```

这个“小样板”跑通后，再批量加入学习内容和练习题。这样未来新增 B5、C、D 单元时，风险会小很多。

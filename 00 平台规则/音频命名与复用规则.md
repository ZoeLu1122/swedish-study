# 音频命名与复用规则

本文档用于管理 ElevenLabs mp3、Excel 映射表、manifest 和页面播放逻辑。

## 1. 三个概念

主文件：

```text
真正保存 mp3 的那条音频。
Excel 中“是否主文件”填“是”。
```

复用文件：

```text
本单元需要这个音频，但不重新生成 mp3。
Excel 中“是否主文件”填“否”。
“沿用的原文件名”填写真正存在的原 mp3。
“复用时的新文件名”填写本单元逻辑上使用的新名字。
```

manifest：

```text
网页实际使用的音频索引。
页面播放时不是直接看 Excel，而是看 audio-manifest.json 和 content-manifest.json。
```

## 2. 文件命名规则

推荐格式：

```text
b1_u3_vocab_001_en_blomma_v1.mp3
b1_u3_dialog_001_line_01_hur_manga_rum_har_din_nya_lagenhet_v1.mp3
b1_u3_qp_001_xxx_v1.mp3
b1_u3_adv_001_xxx_v1.mp3
```

含义：

```text
b1
B 级。

u3
第 3 单元。

vocab
单词页音频。

dialog
对话音频。

qp
练习页基础题音频。

adv
练习页进阶题音频。

v1
第 1 版。
```

## 3. 什么时候复用旧音频

可以复用：

```text
同一个单词或同一句话，发音内容完全一致。
声音性别可以接受。
语速和语气适合当前场景。
```

不建议复用：

```text
内容只相似但不完全相同。
一个是男声、一个是女声，而且当前题型需要固定声音。
单独词和完整短语听感差异很大。
旧音频发音、停顿、语气不适合新页面。
```

## 4. en/ett + 名词的规则

B3 开始采用这个规则：

```text
单词页：en stol、ett hus 等可以播放完整短语。
练习页名词变形：界面仍然可以保留两个空，例如 en + stol。
但音频优先使用完整短语 en stol，而不是 en 和 stol 两个音频拼接。
```

原因：

```text
完整短语更自然。
避免 en/ett 和名词之间听起来断裂。
以后复制到 B4/C/D 更稳定。
```

## 5. Excel 必填规则

新增音频时，至少要填：

```text
级别
单元
板块
语音内容
声音性别
是否主文件
沿用的原文件名
复用时的新文件名
```

主文件示例：

```text
是否主文件：是
沿用的原文件名：留空
复用时的新文件名：b1_u3_adv_085_en_stol_v1.mp3
```

复用文件示例：

```text
是否主文件：否
沿用的原文件名：a1_u2_adv_020_manaden_v1
复用时的新文件名：b1_u3_vocab_042_manaden_v1
```

## 6. overrides 文件

当前 B3 有一个人工修正规则文件：

```text
01 音频整理/b3_unit3_mapping_overrides.json
```

它不是网页运行必需文件，但目前仍建议保留，因为它记录了 B3 中少量人工确认过的复用修正。

长期目标：

```text
把 overrides 里的规则合并回 Excel。
最终让 Excel 成为唯一音频台账。
```

## 7. 删除音频前的判断

可以考虑删除：

```text
Excel 中不建议保留。
manifest 中没有使用。
页面中没有直接引用。
检查工具没有报错。
```

不能删除：

```text
它是主文件。
它被其他单元复用。
它还在 audio-manifest.json 中出现。
它虽然当前单元不用，但未来明确要继续用。
```

## 8. 推荐检查命令

B3：

```bash
python3 "Swedish study/02 tools/check_unit.py" --unit b3 --skip-index
```

导出当前使用音频清单：

```bash
python3 "Swedish study/02 tools/export_used_audio_mapping.py" --unit b3
```

生成预览 manifest：

```bash
python3 "Swedish study/02 tools/build_unit_audio_manifests.py" --unit b3
```

# 新单元制作流程

本文档用于以后新增 B4、B5、C、D 等单元。核心原则：先定数据和音频，再生成清单，最后测试页面。

## 1. 不建议一开始就写页面

新单元不要先复制页面乱改。推荐顺序是：

```text
1. 确认单元主题和内容范围
2. 准备单词、对话、练习题
3. 整理音频映射表
4. 放入 mp3 文件
5. 生成 manifest
6. 跑检查
7. 本地测试
8. 上传 GitHub
9. 线上测试
```

## 2. 新单元需要哪些文件

以 B3 为例：

```text
sfi_b3_unit3.html
sfi_practice_B_unit3.html
b3_unit3_bank.js
audio/sv/b3/unit3/audio-manifest.json
audio/sv/b3/unit3/content-manifest.json
audio/sv/b3/unit3/vocab/*.mp3
audio/sv/b3/unit3/dialogue/*.mp3
audio/sv/b3/unit3/question/*.mp3
```

未来 B4 可以对应：

```text
sfi_b4_unit4.html
sfi_practice_B_unit4.html
b4_unit4_bank.js
audio/sv/b4/unit4/
```

## 3. 先做 Excel，再生成清单

音频总台账是：

```text
01 音频整理/瑞典语学习-音频整理.xlsx
```

新增内容时，先在 Excel 里记录：

```text
级别
单元
板块
语音内容
声音性别
是否主文件
沿用的原文件名
复用时的新文件名
使用来源
是否建议保留
备注
```

不要只把 mp3 放进文件夹，却不更新 Excel。这样后面会不知道文件来源。

## 4. 生成 manifest

manifest 是网页实际查音频用的清单：

```text
audio-manifest.json
content-manifest.json
```

以后生成预览清单时使用：

```bash
python3 "Swedish study/02 tools/build_unit_audio_manifests.py" --unit b3
```

生成后通常会得到：

```text
audio-manifest.preview.json
content-manifest.preview.json
```

确认无误后，再替换正式文件：

```text
audio-manifest.json
content-manifest.json
```

预览文件不要长期上传 GitHub。

## 5. 跑检查

每次改完音频、Excel、manifest、题库后，都跑：

```bash
python3 "Swedish study/02 tools/check_unit.py" --unit b3 --skip-index
```

检查目标：

```text
ERROR=0
没有缺失音频文件
没有 content 找不到 audio 的情况
```

警告不一定都是错误。例如跨单元复用音频时，会出现 cross-unit warning，这是允许的。

## 6. 本地测试

本地测试页面：

```text
http://127.0.0.1:8765/sfi_b3_unit3.html
http://127.0.0.1:8765/sfi_practice_B_unit3.html
```

测试重点：

```text
单词页点击发音正常
单词播放后高亮会消失
练习页基础模式能播放
练习页进阶模式能播放
下一题自动播放正常
切换模式不会播放上一题旧音频
测验页不显示题型标签
```

## 7. 上传 GitHub 后测试

上传后做短测试即可，不需要每次全量测试。

建议每次至少测：

```text
1 个单词页音频
1 个对话页音频
1 个基础练习音频
1 个进阶练习音频
1 个测验页显示效果
```

如果线上和本地不一致，先强制刷新页面。

## 8. 删除文件流程

删除任何 mp3 前：

```text
1. 先从 Excel 中确认它不是主文件
2. 跑导出工具看是否仍在使用
3. 跑检查工具
4. 本地测试
5. 再删除
6. 上传 GitHub
7. 线上复测
```

不要只看文件名判断能不能删。
